Fix PowerShell Execution Policy for Leave Management System

Step 1: Open PowerShell as Administrator

Right-click on the PowerShell icon and select "Run as administrator".

Step 2: Check Current Execution Policy

Run the following command to check your current execution policy:

Get-ExecutionPolicy

Step 3: Set Execution Policy to Allow Scripts

Run one of the following commands based on your security requirements:

Option 1 (Recommended): Allow local scripts but require remote scripts to be signed:

Set-ExecutionPolicy RemoteSigned

Option 2: Allow all scripts (less secure):

Set-ExecutionPolicy Unrestricted

Warning: Setting the execution policy to Unrestricted reduces security. Only use this option if necessary and in a controlled environment.

Step 4: Confirm the Change

Type "Y" and press Enter when prompted to confirm the change.

Step 5: Run the Setup Script Again

After changing the execution policy, try running the setup script again:

.\setup_dual_roles.bat

Or use the development mode script:

.\setup_dev_mode.bat

Alternative: Run for Current Session Only

If you don't want to change the system-wide policy, you can bypass it for the current PowerShell session only:

powershell -ExecutionPolicy Bypass -File .\setup_dual_roles.bat

Alternative: Start the Application Manually

If you're still having issues, you can run these commands directly:

# Import database data
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -proot leave_management < src\db\real_credentials_import.sql

# Start development server
npx next dev
